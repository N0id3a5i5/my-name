# MCQ Exam Simulator

Upload any competitive exam PDF (UPSC, UPSI, SSC, etc.) and practice with a timed MCQ test. AI extracts questions automatically using Groq, Gemini, and/or NVIDIA NIM.

---

## Features

- **Multi-AI parallel extraction** ‚Äî Groq (primary, fast), Gemini (fallback + vision OCR), NVIDIA NIM (optional, +2 workers)
- **Hindi / Devanagari support** ‚Äî handles garbled Kruti Dev / Chanakya fonts via OCR fallback
- **Two-column PDF layout** ‚Äî gap-analysis correctly reads left column before right
- **Scanned PDF OCR** ‚Äî Gemini vision or NVIDIA vision recovers image-only pages
- **Study Mode** ‚Äî instant right/wrong feedback after each selection
- **Question Navigator** ‚Äî jump to any question; flag for review
- **Keyboard shortcuts** ‚Äî A/B/C/D to answer, ‚Üê/‚Üí to navigate, F to flag
- **Custom timer** ‚Äî 30m / 1h / 1.5h / 2h / 3h
- **Retry Wrong** ‚Äî one-tap re-test of only the questions you got wrong
- **Score Analytics** ‚Äî bar chart of attempt history per test
- **SQLite storage** ‚Äî tests and attempt history persist across app restarts (localStorage fallback for web)
- **Save PDF report** ‚Äî export results as a PDF

## Setup (from scratch)

### Prerequisites

- Node.js 18+ and npm
- Android Studio (for local builds) OR push to GitHub (Actions builds the APK for you)
- A free API key from at least one of:
  - [Groq](https://console.groq.com) ‚Äî fastest, recommended
  - [Google AI Studio](https://aistudio.google.com/app/apikey) ‚Äî needed for vision/OCR on scanned PDFs
  - [NVIDIA NIM](https://build.nvidia.com) ‚Äî optional extra workers

### Install and run locally (web)

```bash
npm install
npm run dev
```

Open http://localhost:5173 in your browser.

### Build Android APK locally

```bash
npm install
npm run build

# First time only ‚Äî creates the android/ folder
npx cap add android

# Every time after a code change
npx cap sync android

# Open in Android Studio to build / run on device
npx cap open android
```

Or just push to GitHub ‚Äî the Actions workflow builds the APK automatically.

### Build via GitHub Actions (recommended for phone-only workflow)

1. Push this repo to GitHub (including the `android/` folder after running `npx cap add android` once)
2. Go to **Actions ‚Üí Build Android APK ‚Üí Run workflow**
3. After ~5 minutes, download `MCQ-Simulator-debug.apk` from the Artifacts section
4. Install on your Android device (enable "Install from unknown sources")

## Adding a new AI provider

1. Add a `capFetch` call in a new `xxxCallOne` function following the pattern of `groqCallOne`
2. Add the model to the `aiCallers` array in `startExtraction`
3. Add an API key input field in `renderUpload` following the Groq/NVIDIA pattern
4. Persist the key with `dbSetSetting` / `dbGetSetting`

## Troubleshooting

| Problem | Fix |
|---|---|
| "Failed to fetch" on Android | Make sure CapacitorHttp is enabled in `capacitor.config.ts` and you ran `npx cap sync` |
| Blank questions with no answer | Add a Gemini key ‚Äî it's needed for answer key backfill on scanned PDFs |
| All Groq slices rate-limited | Wait 60 s and retry, or add a Gemini/NVIDIA key as fallback |
| Hindi text garbled | Add a Gemini key ‚Äî OCR vision mode will recover the pages |
| APK build fails in Actions | Check Java version is 17 and `android/gradlew` is committed as executable |
